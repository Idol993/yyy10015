## 1. 产品概述

本项目实现一个基于WebGPU的无偏实时路径追踪渲染器，完全运行在浏览器中，不使用任何光栅化替代方案，所有渲染工作由GPU计算着色器完成。面向专业图形开发者和实时渲染研究人员，提供可交互的高质量光线追踪体验。

目标是在消费级GPU（NVIDIA RTX 3060）上实现1280×720分辨率下20+FPS的交互渲染，32样本累积后接近收敛的图像质量。

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|-----------|------------|
| 开发者 | 无需注册，直接访问 | 加载自定义glTF场景、调节渲染参数、查看性能剖析数据 |
| 研究者 | 无需注册，直接访问 | 调整高级渲染算法参数、导出渲染结果 |

### 2.2 功能模块

1. **主渲染视图**：实时路径追踪渲染画布、性能统计叠加层
2. **场景管理面板**：glTF场景加载器、材质参数编辑器、场景层级浏览
3. **渲染参数面板**：采样数、最大深度、降噪参数调节
4. **后处理面板**：曝光、色调映射、Bloom、景深参数
5. **相机控制面板**：自由相机控制、景深参数

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|-----------|-----------|----------------|
| 主渲染页面 | 渲染画布 | WebGPU Canvas实时渲染输出，支持拖拽旋转、滚轮缩放 |
| 主渲染页面 | 性能统计 | FPS、各Pass耗时、GPU内存使用实时显示 |
| 主渲染页面 | 控制面板 | 可折叠的参数调节侧边栏 |
| 主渲染页面 | 场景加载 | 内置示例场景选择、自定义glTF文件上传 |

## 3. 核心流程

用户打开页面 → 初始化WebGPU设备 → 加载默认场景 → 构建BVH加速结构 → 实时路径追踪渲染循环 → 降噪后处理 → 色调映射输出 → 用户交互（相机移动/参数调整 → 动态重建BVH（如需）

```mermaid

    G --> H["输出到Canvas"]
    H --> I["用户交互"]

    J --> K["场景更新"]
    K --> D
```

## 4. 用户界面设计

### 4.1 设计风格
- **设计风格：科技暗黑科技感，深色主题，霓虹蓝绿色调
- **主色调**：#0a0a0f 深黑背景，#00d4ff 青色强调色，#7c3aed 紫色点缀
- **按钮样式**：玻璃拟态圆角按钮，悬浮发光效果
- **字体**：JetBrains Mono 等宽字体用于技术感
- **布局风格**：全屏渲染区域 + 右侧可折叠侧边栏，半透明玻璃效果
- **图标风格**：线性简约线性图标，霓虹发光效果

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|-----------|-----------|----------|
| 主渲染页面 | 渲染画布 | 全屏Canvas，右上角性能统计浮层 |
| 主渲染页面 | 侧边栏 | 半透明玻璃面板，分组折叠，滑动条，下拉选择器 |
| 主渲染页面 | 性能面板 | 实时折线图显示各Pass耗时 |
| 主渲染页面 | 加载状态 | WebGPU初始化进度条，动画过渡效果 |

### 4.3 响应式
- 桌面端优先，侧边栏在移动端自动隐藏，渲染区域自适应窗口大小

### 4.4 3D场景指引
- **环境/HDRI**：内置多个HDR环境贴图，支持用户自定义上传
- **光照**：物理正确的PBR光照，支持HDR环境光重要性采样
- **相机**：自由飞行相机，支持WASD移动，鼠标拖拽旋转
- **后处理**：ACES电影级色调映射，Bloom光晕效果，景深模糊，景深Bokeh
## 5. 非功能需求

### 5.1 性能需求
- RTX 3060级别GPU上1280×720分辨率下≥20FPS
- 50万三角形BVH构建≤1秒
- 32样本累积后图像接近收敛
- 内存占用≤2GB GPU显存

### 5.2 兼容性需求
- 支持WebGPU的现代浏览器（Chrome 113+，Edge 113+，Safari 16.4+）
- 降级提示不支持WebGPU的浏览器

### 5.3 可扩展性
- 模块化设计，易于扩展新的材质模型和降噪算法
